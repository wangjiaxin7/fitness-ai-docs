# AI 健身教练 — 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品定位

AI 健身教练是一款面向个人健身爱好者的 Web 应用，通过 AI 对话、动作视频分析和数据驱动的训练建议，解决"没人指导"的核心痛点。

### 1.2 目标用户

**主要用户画像**：有一定健身基础（3 个月以上）但缺乏专业指导的健身爱好者

| 用户特征 | 描述 |
|---------|------|
| 年龄 | 20-35 岁 |
| 健身经验 | 3 个月 - 3 年，知道基本动作但不确定做得对不对 |
| 痛点 | 训练计划不知道怎么调整、动作做得对不对没人看、训练数据没有系统分析 |
| 场景 | 商业健身房、家庭健身 |
| 付费意愿 | 中等，愿意为专业指导付费但不想请私教（太贵） |

### 1.3 核心价值主张

1. **看得见的动作评估**：上传训练视频，AI 逐帧分析关节角度，给出多维度评分和改进建议
2. **问得到的专业指导**：AI 对话教练基于专业知识库回答训练问题，不是泛聊天机器人
3. **用得越久越精准**：训练数据持续积累，AI 建议从通用走向个性化

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 用户留存 | 7 日留存 > 40% | 登录行为统计 |
| 动作分析使用率 | 活跃用户 > 60% 使用视频分析 | 功能埋点 |
| 对话质量 | 用户满意度 > 80% | 对话后反馈评分 |
| 训练频率 | 使用产品后训练频率提升 | 训练记录对比 |

### 2.2 功能详细设计

#### 2.2.1 用户系统

**需求描述**：用户注册、登录、个人信息管理

**功能点**：
- 用户名 + 密码注册（密码 SHA256 加盐存储）
- 用户名 + 密码登录（Session 管理）
- 个人档案管理：姓名、身高、体重、年龄、训练目标、经验水平、可用器材
- 管理员角色（可查看所有用户数据）

**验收标准**：
- [x] 注册后自动登录
- [x] 密码加密存储，不明文保存
- [x] Session 过期后自动跳转登录页
- [x] 个人档案信息在 AI 对话中自动注入上下文

#### 2.2.2 AI 对话教练

**需求描述**：基于大模型的智能健身对话，具备专业知识和安全意识

**功能点**：
- 流式对话（SSE），实时输出打字效果
- 四层上下文注入：系统提示词 + RAG 检索 + 对话历史 + 用户档案
- 对话历史管理（新建对话、切换对话、历史记录）
- 安全预检：伤病关键词检测 → 直接引导就医，不调用 LLM
- RAG 知识检索：31 篇健身文档，向量化后余弦相似度检索 Top-3

**对话流程**：
```
用户输入
  ↓
伤病关键词预检 ──→ 命中 → 返回就医建议（不调 LLM）
  ↓ 未命中
RAG 知识检索（Top-3 相关文档）
  ↓
构建 Prompt（系统提示词 + RAG 结果 + 对话历史 + 用户档案）
  ↓
调用 DeepSeek API（流式 SSE）
  ↓
返回前端实时展示
  ↓
保存对话到记忆服务
```

**验收标准**：
- [x] 对话响应延迟 < 3 秒（首 token）
- [x] 伤病问题不给出训练建议，直接引导就医
- [x] RAG 检索结果相关性 > 80%
- [x] 对话历史可切换和管理

#### 2.2.3 动作视频分析

**需求描述**：用户上传训练视频，AI 逐帧分析动作质量并给出评分和建议

**支持动作**：

| 动作 | 评分维度 |
|------|---------|
| 深蹲 | 膝盖内扣、下蹲幅度、躯干前倾、节奏、稳定性 |
| 卧推 | 杠铃轨迹、肘部角度、幅度、节奏 |
| 引体向上 | 幅度、身体摆动、对称性、节奏 |
| 硬拉 | 背部曲度、髋铰链幅度、杠铃轨迹、节奏 |
| 杠铃划船 | 躯干角度、幅度、稳定性、对称性 |
| 二头弯举 | 肘部稳定性、幅度、节奏、惯性控制 |
| 腿举 | 幅度、膝盖内扣、节奏、稳定性 |

**分析流程**：
```
视频上传
  ↓
逐帧 MediaPipe 姿态估计（33 个关键点）
  ↓
关键关节角度计算（肘角、膝角、髋角等）
  ↓
角度序列分析 → 各维度评分（0-10）
  ↓
综合评分 + 改进建议
  ↓
调用 DeepSeek 生成教练点评
  ↓
返回评分、图表、点评
```

**关键设计决策**：
- 用**肘角**替代 depth ratio，实现与拍摄角度无关的评分
- 2D 视频即可，不要求 3D 深度信息
- 本地计算，不需要 GPU

**验收标准**：
- [x] 视频上传后 30 秒内返回分析结果
- [x] 评分与专业教练判断一致性 > 80%
- [x] 支持常见手机拍摄角度（正面、侧面 45°）
- [x] 分析结果包含可视化角度变化图表

#### 2.2.4 训练记录管理

**需求描述**：记录每次训练的详细数据

**功能点**：
- 记录训练动作、组数、次数、重量
- 按日期查看训练历史
- 训练后反馈（疲劳度、疼痛、满意度）
- 数据导出

**验收标准**：
- [x] 记录一条训练 < 10 秒
- [x] 支持按日期和动作筛选
- [x] 训练数据持久化存储

#### 2.2.5 训练决策引擎

**需求描述**：基于用户历史数据，自动生成训练建议

**三层架构**：
1. **规则引擎**：36 条硬编码规则，覆盖行为、安全、状态三个维度
2. **趋势分析**：分析训练频率、负荷变化、身体指标趋势
3. **LLM 解释**：将规则输出用自然语言生成可读建议

**冷启动策略**：

| 阶段 | 训练次数 | 策略 |
|------|---------|------|
| 引导期 | 0 次 | 引导用户完成首次训练 |
| 探索期 | 1-3 次 | 基于有限数据给出基础建议 |
| 建模期 | 4-7 次 | 开始分析训练模式和偏好 |
| 成熟期 | 8+ 次 | 完整的个性化建议 |

**验收标准**：
- [x] 规则引擎覆盖常见训练场景
- [x] 建议可读、可执行，不是空话
- [x] 冷启动阶段有合理的引导策略

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 |
|------|------|
| AI 对话首 token 响应 | < 3 秒 |
| 视频分析完成时间 | < 30 秒 |
| 页面加载时间 | < 2 秒 |
| 并发用户数 | > 50 |

### 3.2 安全

- 密码 SHA256 加盐存储
- API Key 环境变量管理，不硬编码
- SQL 参数化查询，防止注入
- XSS 输入过滤和转义
- Session HttpOnly + Secure

### 3.3 可用性

- 7×24 小时可用
- 容器自动重启（unless-stopped）
- 健康检查端点

## 4. 用户故事

### 4.1 核心用户故事

| ID | 用户故事 | 验收标准 | 优先级 |
|----|---------|---------|--------|
| US-01 | 作为健身爱好者，我想上传深蹲视频，让 AI 告诉我哪里做得不好 | 上传视频后返回评分、改进建议、教练点评 | P1 |
| US-02 | 作为健身爱好者，我想问 AI 教练"卧推肩膀疼怎么办"，得到专业建议 | 检测到伤病关键词，引导就医而非给训练建议 | P0 |
| US-03 | 作为健身爱好者，我想让 AI 根据我的训练历史推荐今天的训练内容 | 基于历史数据生成个性化建议 | P2 |
| US-04 | 作为健身爱好者，我想看自己这一个月的训练趋势 | 数据看板展示训练频率、负荷变化趋势 | P2 |
| US-05 | 作为健身爱好者，我想让 AI 教练知道我的身高体重和训练目标 | 个人档案信息自动注入对话上下文 | P0 |

### 4.2 边界场景

| 场景 | 处理方式 |
|------|---------|
| 用户上传非健身视频 | MediaPipe 无法检测人体关键点 → 返回"未检测到人体"提示 |
| 用户询问伤病问题 | 关键词预检命中 → 返回就医建议，不调用 LLM |
| DeepSeek API 超时 | 30 秒超时 → 返回"服务繁忙"提示 |
| 用户训练数据不足 | 冷启动策略 → 给出通用建议 + 鼓励继续训练 |
