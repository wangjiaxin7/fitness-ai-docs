# 数据库设计

## 1. 概述

- **数据库**：PostgreSQL 14+
- **表数量**：6 张自定义业务表
- **设计原则**：用户数据按 user_id 隔离，高频查询建索引，JSON 字段存储灵活数据

## 3. 表结构

### 3.1 web_users — 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID (PK) | 用户唯一标识 |
| username | VARCHAR(50) | 用户名（唯一） |
| password_hash | VARCHAR(128) | 密码哈希（SHA256 + salt） |
| display_name | VARCHAR(100) | 显示名称 |
| is_admin | BOOLEAN | 是否管理员 |
| created_at | TIMESTAMP | 注册时间 |
| last_login | TIMESTAMP | 最后登录时间 |

**索引**：
- `idx_web_users_username` — username 唯一索引

**密码存储格式**：`salt:hash`（salt 为 32 位十六进制随机字符串）

### 3.2 training_records — 训练记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID (PK) | 记录唯一标识 |
| user_id | UUID (FK) | 关联用户 |
| exercise_name | VARCHAR(100) | 动作名称 |
| sets | INTEGER | 组数 |
| reps | INTEGER | 每组次数 |
| weight | DECIMAL(6,2) | 重量（kg） |
| duration | INTEGER | 时长（分钟） |
| notes | TEXT | 备注 |
| created_at | TIMESTAMP | 记录时间 |

**索引**：
- `idx_training_records_user_id` — user_id
- `idx_training_records_created_at` — created_at
- `idx_training_records_user_exercise` — (user_id, exercise_name)

**典型查询**：
```sql
-- 查询用户最近 30 天的训练记录
SELECT * FROM training_records
WHERE user_id = %s AND created_at > NOW() - INTERVAL '30 days'
ORDER BY created_at DESC;

-- 统计各动作的训练频次
SELECT exercise_name, COUNT(*) as count
FROM training_records
WHERE user_id = %s
GROUP BY exercise_name
ORDER BY count DESC;
```

### 3.3 body_measurements — 身体测量

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID (PK) | 记录唯一标识 |
| user_id | UUID (FK) | 关联用户 |
| weight | DECIMAL(5,2) | 体重（kg） |
| body_fat | DECIMAL(4,1) | 体脂率（%） |
| chest | DECIMAL(5,1) | 胸围（cm） |
| waist | DECIMAL(5,1) | 腰围（cm） |
| hip | DECIMAL(5,1) | 臀围（cm） |
| bicep_left | DECIMAL(5,1) | 左臂围（cm） |
| bicep_right | DECIMAL(5,1) | 右臂围（cm） |
| thigh_left | DECIMAL(5,1) | 左腿围（cm） |
| thigh_right | DECIMAL(5,1) | 右腿围（cm） |
| measured_at | TIMESTAMP | 测量时间 |

**索引**：
- `idx_body_measurements_user_id` — user_id
- `idx_body_measurements_measured_at` — measured_at

### 3.4 training_feedback — 训练反馈

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID (PK) | 记录唯一标识 |
| user_id | UUID (FK) | 关联用户 |
| record_id | UUID (FK) | 关联训练记录（可选） |
| fatigue_level | INTEGER | 疲劳度（1-10） |
| pain_level | INTEGER | 疼痛度（1-10，0=无痛） |
| satisfaction | INTEGER | 满意度（1-10） |
| notes | TEXT | 备注 |
| created_at | TIMESTAMP | 反馈时间 |

### 3.5 decision_logs — 决策日志

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID (PK) | 日志唯一标识 |
| user_id | UUID (FK) | 关联用户 |
| decision_type | VARCHAR(50) | 决策类型（规则/趋势/LLM） |
| trigger | VARCHAR(100) | 触发条件 |
| recommendation | TEXT | 建议内容 |
| confidence | DECIMAL(3,2) | 置信度 |
| created_at | TIMESTAMP | 决策时间 |

### 3.6 training_plans — 训练计划

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID (PK) | 计划唯一标识 |
| user_id | UUID (FK) | 关联用户 |
| plan_name | VARCHAR(100) | 计划名称 |
| exercises | JSONB | 训练内容（JSON） |
| status | VARCHAR(20) | 状态（active/completed/archived） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

```json
    "sets": 4,

  }
]
```

## 4. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 主键类型 | UUID | 分布式友好，不暴露用户数量 |
| 密码存储 | SHA256 + 随机 salt | 比 bcrypt 实现简单，安全性足够 |
| JSON 字段 | JSONB（training_plans.exercises） | 训练计划结构灵活，不适合固定列 |
| 时间字段 | TIMESTAMP | 支持时区，便于跨区域部署 |
| 软删除 | 未采用 | 当前规模不需要，直接硬删除 |

## 5. 索引策略

- 所有外键字段建索引（user_id）
- 高频查询的排序字段建索引（created_at）
- 复合查询建复合索引（user_id + exercise_name）
- 不过度索引：写入频繁的表控制索引数量
